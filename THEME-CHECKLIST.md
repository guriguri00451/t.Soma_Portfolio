# ポートフォリオ制作用 AI-DLCチェックリスト

* 前提：GitHub Pagesでの一般公開を目指し、AI-DLCのループを回して段階的に作り上げる。
* ゴールは「自身のスキルや実績を適切にアピールできる、保守しやすいポートフォリオを完成させること」。
* 技術スタックは静的サイトホスティング（GitHub Pages）に適合するものを選ぶ。

## A. スコープ（段階的な構築とイテレーション）

* [ ] 「自己紹介」「作品一覧」「コンタクト」など、セクションごとに分割してAI-DLCの計画・実装ループを回せる粒度になっているか
* [ ] 最初から完璧なデザインを目指さず、まずは「テキストと画像が並んだ最小限の形（MVP）」を作り、後からスタイルを調整する計画になっているか

## B. 技術・環境（GitHub Pagesでの公開前提）

* [ ] サーバーサイドの動的な処理（DBやPHPなど）に依存せず、静的サイトジェネレーター（React/Next.jsの静的エクスポート等）で完結する構成になっているか
* [ ] お問い合わせ機能などが必要な場合、バックエンド不要な外部サービス（Googleフォームやformspreeなど）を利用する想定になっているか

## C. 設計・判断の余地（見せ方と構成） ← 重要

* [ ] 誰（採用担当者、クライアント、技術仲間など）に向けて、何を一番アピールしたいポートフォリオなのか（Intent）が明確か
* [ ] 単なるテンプレートの丸写しではなく、「どの実績をどういう順番で載せるか」「どのようなUI/UXで回遊させるか」についてAIに提案させ、人間が取捨選択する余地があるか

## D. 安全性と公開リスク（権利とプライバシー）

* [ ] 掲載する作品やプロジェクトについて、外部に公開する権利（NDAや共同制作者の許諾など）がクリアになっているか
* [ ] 自身の経歴や連絡先に、公開すべきでない過度な個人情報（詳細な住所やプライベートな電話番号など）が含まれていないか
* [ ] 使用する画像、フォント、ライブラリのライセンスは商用/個人公開で問題ないものか

## E.（任意）モチベーション・拡張性

* [ ] 今後、新しい作品ができたときに更新しやすい（またはAIに更新を頼みやすい）ファイル構造・データ構造になっているか
* [ ] 自分が作っていて「テンションが上がる」デザインや技術要素を1つは取り入れているか

---

## 判定の勘所

* A〜Dが概ね満たせればOK。
* Cの「誰に何をアピールしたいか」がブレるとAIの提案もブレるため、AI-DLCを回す最初の「要件定義（Requirements Analysis）」フェーズで、AIとしっかり壁打ちして言語化して欲しい。
