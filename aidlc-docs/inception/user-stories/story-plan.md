## 作成するストーリー一覧（計画）

### P1: インターン採用担当向け

- [ ] US-01: エントリー書類からポートフォリオ URL を開き、候補者の第一印象を掴む
- [ ] US-02: スキルセットを見て、自社の技術スタックと合うか確認する
- [ ] US-03: 制作物を見て、実際に何を作れるか具体的に把握する
- [ ] US-04: GitHub リンクからコードの書き方・活動量を確認する

### P2: サイトオーナー向け

- [ ] US-05: 新しいプロジェクトが完成したとき、実績セクションに追加する
- [ ] US-06: インターン選考が終わったあと、Phase 2 コンテンツ（職歴・記事）を追加する

### P3: 技術コミュニティ向け

- [ ] US-07: SNS 経由でプロフィールに辿り着き、技術スタックと制作物を確認する
- [ ] US-08: 興味深いプロジェクトを見つけ、GitHub リポジトリへ移動する

## 各ストーリーに含める項目

- **タイトル**: 「〜として、〜したい。なぜなら〜だから」形式
- **前提条件 (Given)**
- **操作 (When)**
- **期待結果 (Then)**
- **受け入れ基準**（検証可能な条件）
